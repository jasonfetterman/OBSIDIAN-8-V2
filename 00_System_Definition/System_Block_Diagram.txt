1. PURPOSE

for OBSIDIAN‑8 V3 with integrated Duet 3 motion system.

------------------------------------------------------------
3. PERCEPTION FLOW
------------------------------------------------------------

Sensors (Camera, Depth, IMU, Environmental) feed → Perception Stack → Mission.  
Perception outputs pose and terrain data only—no direct actuator commands.  
Reference: /05_Perception/Sensor_Suite_Design.txt  

------------------------------------------------------------
4. POWER FLOW ARCHITECTURE
------------------------------------------------------------

                 |  Battery Pack    |
                 |  + Smart BMS     |
                 +--------+---------+

Kill‑Line Relay between Power Distribution and Servo Rail.  
Hardware E‑Stop interrupts Servo Rail directly.  

------------------------------------------------------------
5. SAFETY SUPERVISION BLOCK
------------------------------------------------------------

            +-----------------------+
            | Pi Watchdog System    |
            | – Heartbeat Monitor   |
            | – Authority Check     |
            +-----------+-----------+
                        |
                        v
                Kill‑Line Relay
                        |
                        v
               Duet 3 Enable Input

Watchdog supervises:
- Reactive Controller  
- Mission Compute  
- duet_bridge heartbeat  

On loss of communication → Kill‑Line release (open) → Servo rail disables.  
Reference: /11_Codebase/pi_watchdog/  

------------------------------------------------------------
6. AUTHORITY FLOW
------------------------------------------------------------

Layer 4 – Maintenance Mode  
Layer 3 – Remote Supervision  
Layer 2 – Mission Compute  
Layer 1 – Reactive Controller  
Layer 0 – Hardware Kill (Watchdog + Duet chain)  

Authority flows downward, safety dominance flows upward.  

Duet 3 board executes motion commands issued from Layer 1 but power is
governed by Layer 0. No Duet motion may occur without active heartbeat
validation from Reactive and Watchdog.  

------------------------------------------------------------
7. DATA FLOW SUMMARY
------------------------------------------------------------

Sensors → Perception → Mission → Reactive → duet_bridge → Duet 3 → Actuators  
Remote → Mission → Reactive → Duet 3 → Actuators  

Watchdog monitors Reactive, Mission, and duet_bridge rates.  
Hardware Kill cuts Servo power independent of software state.  

------------------------------------------------------------
8. FAILURE CONTAINMENT BOUNDARIES
------------------------------------------------------------

Mission failure → Reactive STOP  
Reactive failure → Watchdog Kill‑Line  
duet_bridge failure → Duet heartbeat timeout → STOP  
Duet firmware fault → Kill‑Line open → Servo disable  
Watchdog failure → Kill‑Line defaults open  
BMS fault → Power Isolation  

No single failure may result in uncontrolled motion.  
Reference: /00_System_Definition/Failure_Modes_and_Effects_Analysis.txt  

------------------------------------------------------------
9. DESIGN INTENT
------------------------------------------------------------

Architecture ensures:  
- Deterministic authority enforcement  
- Hardware‑dominant safety cutoff  
- Transparent data and power pathways  
- Isolated perception and actuation domains  
- Integrated Duet 3 motion control within safety hierarchy  

This diagram is the canonical architecture for OBSIDIAN‑8 V3 with Duet 3.

FM‑014  
Subsystem: Duet 3 Bridge Interface  
Failure Mode: Loss of duet_bridge heartbeat or USB link  
Cause: Software crash / communication timeout  
Effect: Servo halt / motion freeze  
S: 4  O: 2  D: 2  RPN: 16  
Mitigation: Reactive STOP + Watchdog Kill‑Line trigger  
Reference: /02_Electrical/Duet3_Interface.txt  

------------------------------------------------------------

FM‑015  
Subsystem: Duet 3 Mainboard  
Failure Mode: Firmware fault or overcurrent shutdown  
Cause: Internal firmware exception / power surge  
Effect: Servo power loss / reset  
S: 3  O: 2  D: 3  RPN: 18  
Mitigation: BMS current limiting + automatic re‑initialization sequence  
Reference: /02_Electrical/Duet3_Interface.txt  

------------------------------------------------------------

FM‑016  
Subsystem: Duet 3 Temperature Sensor Channel  
Failure Mode: False reading causing premature shutdown  
Cause: Sensor offset or EMI disturbance  
Effect: Unexpected servo disable, reduced mobility  
S: 2  O: 3  D: 3  RPN: 18  
Mitigation: Sensor averaging + redundant thermal check through Reactive node  
Reference: /01_Mechanical/Actuation_Systems/Thermal_Model.txt  

------------------------------------------------------------

FM‑017  
Subsystem: Duet 3 CAN Bus  
Failure Mode: Bus fault or desync  
Cause: Connector loosening / noise / packet loss  
Effect: Peripheral data loss / debug channel failure  
S: 2  O: 2  D: 3  RPN: 12  
Mitigation: Physical shielding + retransmission protocol check  
Reference: /02_Electrical/Duet3_Interface.txt
