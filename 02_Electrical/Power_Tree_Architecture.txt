Power Tree Architecture — REV B (150KG Servo Integration)

OVERVIEW
This document defines the structured distribution of electrical power throughout OBSIDIAN-8. It ensures stable delivery to high-current servo systems and regulated supply to sensitive control electronics.

--------------------------------------------------

ARCHITECTURE SUMMARY

The system uses a hierarchical power distribution model with two isolated rails:

1. High-Current Servo Rail (7.4V)
2. Regulated Logic Rail (5V)

Both rails originate from a single battery source and share a common ground.

--------------------------------------------------

LEVEL 0 — POWER SOURCE

Component:
- 2S LiPo Battery (7.4V nominal)

Output Path:
Battery
 → Master Power Switch
 → Main Fuse (60A–80A)
 → Central Power Distribution Block

This is the root of the power tree.

--------------------------------------------------

LEVEL 1 — PRIMARY DISTRIBUTION

From the central distribution block, power splits into:

A. Servo Power Bus (Unregulated 7.4V)
B. Logic Power Converter Input (to 5V regulator)

Each branch is electrically independent except for shared ground.

--------------------------------------------------

LEVEL 2A — SERVO POWER DISTRIBUTION

Servo Bus Characteristics:
- High current
- Unregulated voltage
- Noise-generating load

Distribution Method:
- Star topology from central block

Segmentation:
- Divide servos into clusters of 2–4 units

Per-Cluster Design:
- Inline fuse (15A typical)
- Bulk capacitor (2200µF–4700µF)
- Dedicated power leads (16–18 AWG)

Wiring:
- Main feed: 12 AWG
- Keep all paths short and direct

Constraints:
- No daisy-chaining between servos
- Each cluster returns directly to distribution block

--------------------------------------------------

LEVEL 2B — LOGIC POWER DISTRIBUTION

Input:
- 7.4V from main distribution block

Conversion:
- High-efficiency buck converter

Output:
- 5V regulated rail

Capacity:
- Minimum 5A continuous

Supplies:
- Main compute unit (Raspberry Pi or equivalent)
- Sensors
- Communications hardware

Design Requirements:
- Electrically isolated from servo noise
- Stable under all load conditions

--------------------------------------------------

LEVEL 3 — FINAL DISTRIBUTION

Servo Nodes:
- Individual servos connected to cluster outputs

Logic Nodes:
- Distributed via regulated 5V rail
- Can use secondary distribution board if needed

Grounding:
- All grounds return to central distribution point

--------------------------------------------------

GROUNDING MODEL

- Single-point ground reference at distribution block
- All subsystems must share this ground
- Avoid multiple return paths to prevent noise and instability

--------------------------------------------------

NOISE AND INTERFERENCE CONTROL

Servo systems introduce electrical noise.

Mitigation:
- Use bulk capacitors at each servo cluster
- Keep signal lines separate from power lines
- Use twisted pair or shielded cables for control signals where possible

--------------------------------------------------

PROTECTION STRATEGY

Primary Protection:
- Main fuse at battery output

Secondary Protection:
- Per-cluster servo fuses
- Regulator protection for logic rail

Recommended:
- Transient voltage suppression (TVS diodes) on main bus
- Reverse polarity protection at battery input

--------------------------------------------------

FAILURE ISOLATION

Design Goal:
- A failure in one branch must not bring down the entire system

Implementation:
- Independent fused servo clusters
- Isolated logic regulator path

Example:
- If one servo cluster fails, remaining clusters and logic system continue operating

--------------------------------------------------

DESIGN CONSTRAINTS

- Design for peak current, not average load
- Maintain at least 30% current headroom
- Use only connectors rated for expected load
- Keep high-current paths as short as possible
