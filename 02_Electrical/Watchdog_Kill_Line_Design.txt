Watchdog Kill Line Design — REV B (High-Torque Safety Integration)

OVERVIEW
This document defines the emergency shutdown and watchdog system for OBSIDIAN-8. The system is designed to immediately disable actuator power in the event of control failure, software crash, or unsafe operating conditions.

--------------------------------------------------

SYSTEM PURPOSE

The kill system must:
- Instantly cut power to all servos
- Preserve power to control electronics when possible
- Operate independently of high-level software
- Default to a safe state on failure

--------------------------------------------------

ARCHITECTURE SUMMARY

The system consists of two layers:

1. Software Watchdog (Teensy + Raspberry Pi)
2. Hardware Kill Line (Physical power cutoff)

Both systems operate together but independently.

--------------------------------------------------

LAYER 1 — SOFTWARE WATCHDOG

Raspberry Pi:
- Sends continuous heartbeat signal to Teensy (50–100 Hz)

Teensy:
- Monitors heartbeat timing
- If heartbeat is lost:
  → Stops sending PWM updates
  → Holds or neutralizes servo positions

Timeout:
- 100–200 ms maximum before fail-safe triggers

--------------------------------------------------

LAYER 2 — HARDWARE KILL LINE

Primary Function:
- Physically disconnect servo power rail

Implementation Options:

Option A (Recommended):
- High-current relay or solid-state relay (SSR)

Option B:
- MOSFET-based high-side switch rated for full system current

Control Input:
- Driven by Teensy watchdog output pin

Default State:
- OFF (no servo power) unless actively enabled

--------------------------------------------------

CONTROL LOGIC

Normal Operation:
- Teensy receives valid commands
- Watchdog signal remains active
- Kill line remains CLOSED (power enabled)

Failure Conditions:
- Loss of communication from Pi
- Teensy firmware crash or reset
- Detected fault condition

Result:
- Kill signal deactivates
- Relay/MOSFET opens
- Servo power is immediately cut

--------------------------------------------------

POWER ISOLATION

- Kill line affects ONLY servo power rail (7.4V)
- Logic power (5V) remains active to allow recovery and diagnostics

--------------------------------------------------

MANUAL EMERGENCY STOP

Required:
- Physical emergency stop button

Behavior:
- Immediately interrupts kill line control signal
- Forces servo power OFF regardless of software state

Placement:
- Easily accessible on robot exterior

--------------------------------------------------

FAIL-SAFE DESIGN

System must default to SAFE (power OFF) if:
- Control signal wire is disconnected
- Teensy loses power
- System resets unexpectedly

This requires:
- Active-high enable logic (power ON only when signal is present)

--------------------------------------------------

RESPONSE TIME

Target:
- <50 ms from fault detection to power cutoff

Rationale:
- Prevent damage from runaway motion or control instability

--------------------------------------------------

COMPONENT REQUIREMENTS

Switching Device:
- Must support 80A peak current minimum

Relay Option:
- Automotive high-current relay (fast switching preferred)

MOSFET Option:
- Logic-level gate
- Proper heat sinking required
- Low Rds(on)

--------------------------------------------------

FAILURE MODES

Without kill system:
- Runaway servos
- Mechanical damage
- Electrical overload
- Safety hazard

With kill system:
- Controlled shutdown
- Power isolation
- System remains recoverable

--------------------------------------------------

TESTING REQUIREMENTS

Must validate:

1. Communication loss test
2. Teensy reset test
3. Manual E-stop activation
4. High-load shutdown behavior

System must cut servo power reliably in all cases

--------------------------------------------------

SYSTEM CONSTRAINTS

- Kill system must not depend on Raspberry Pi
- Must function even during partial system failure
- Must handle full current without overheating
