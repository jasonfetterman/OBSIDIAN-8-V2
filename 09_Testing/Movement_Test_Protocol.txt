Movement Test Protocol — REV B (High-Torque Quadruped Validation)

OVERVIEW
This document defines the procedures for validating movement performance of OBSIDIAN-8 using 150KG-class servos. The goal is to verify stability, control accuracy, power behavior, and safety under real operating conditions.

--------------------------------------------------

TEST OBJECTIVES

- Verify stable standing and walking behavior
- Validate motion control and smoothness
- Measure power and thermal performance
- Confirm safety systems function correctly
- Identify mechanical or control weaknesses

--------------------------------------------------

TEST ENVIRONMENT

Surface:
- Flat, rigid surface (initial testing)
- Non-slip material preferred

Conditions:
- Indoor environment
- Controlled lighting and temperature

Safety:
- Clear area around robot
- Emergency stop accessible at all times

--------------------------------------------------

PRE-TEST CHECKLIST

- All fasteners tightened
- Wiring secured and inspected
- Battery fully charged
- Kill switch verified operational
- Communication between Pi and Teensy confirmed
- Servo calibration checked (neutral positions)

Do not proceed if any issue is found.

--------------------------------------------------

TEST 1 — STATIC STABILITY

Procedure:
- Power system ON
- Command neutral stance (all legs grounded)
- Observe for 60 seconds

Pass Criteria:
- No visible drift or movement
- No jitter in servos
- No resets or power instability

Fail Conditions:
- Servo oscillation
- System reset
- Audible strain or instability

--------------------------------------------------

TEST 2 — SINGLE LEG LOAD TEST

Procedure:
- Lift one leg slightly off ground (manual or controlled)
- Maintain position for 10–15 seconds
- Repeat for all legs

Pass Criteria:
- Remaining legs maintain stable support
- No excessive tilt or collapse

Fail Conditions:
- Loss of balance
- Servo overload or shutdown

--------------------------------------------------

TEST 3 — CONTROLLED MOTION (NO LOAD)

Procedure:
- Command slow joint movements (one servo at a time)
- Sweep full range within limits

Pass Criteria:
- Smooth motion with no jerks
- Accurate positioning
- No abnormal noise

Fail Conditions:
- Sudden jumps
- Missed positions
- Mechanical binding

--------------------------------------------------

TEST 4 — GAIT VALIDATION (SLOW MODE)

Procedure:
- Enable gait engine (slow mode)
- Run continuous walking cycle for 2 minutes

Pass Criteria:
- Stable alternating gait
- No tipping or slipping
- Smooth transitions between phases

Fail Conditions:
- Loss of balance
- Uneven stepping
- Excessive vibration

--------------------------------------------------

TEST 5 — POWER LOAD TEST

Procedure:
- Run walking cycle under moderate speed
- Monitor voltage and current (if available)

Pass Criteria:
- No system resets
- Stable operation throughout test

Fail Conditions:
- Brownouts
- Controller reboot
- Sudden shutdown

--------------------------------------------------

TEST 6 — THERMAL TEST

Procedure:
- Operate system continuously for 5–10 minutes
- Monitor:
  - Servo temperature
  - Battery temperature
  - Power components

Pass Criteria:
- Components remain within safe temperature range
- No thermal shutdown

Fail Conditions:
- Overheating
- Performance degradation

--------------------------------------------------

TEST 7 — EMERGENCY STOP VALIDATION

Procedure:
- While robot is moving, activate kill switch

Expected Result:
- Immediate cutoff of servo power
- No continued motion

Pass Criteria:
- Response time < 50 ms
- System enters safe state

Fail Conditions:
- Delayed response
- Partial shutdown
- Continued movement

--------------------------------------------------

TEST 8 — COMMUNICATION LOSS TEST

Procedure:
- Interrupt communication between Pi and Teensy

Expected Behavior:
- Servos hold position or enter safe state
- No erratic movement

Pass Criteria:
- Stable behavior during signal loss

Fail Conditions:
- Sudden movement
- Loss of control

--------------------------------------------------

DATA RECORDING

For each test:
- Record pass/fail
- Note any anomalies
- Capture video if possible
- Log voltage/temperature if sensors available

--------------------------------------------------

TEST SEQUENCE

Recommended order:
1. Static Stability
2. Single Leg Load
3. Controlled Motion
4. Gait (Slow)
5. Power Load
6. Thermal
7. Emergency Stop
8. Communication Loss

Do not proceed to next test if current test fails.

--------------------------------------------------

ACCEPTANCE CRITERIA

System is considered validated when:
- All tests pass without critical failure
- No repeated instability observed
- No component damage occurs
